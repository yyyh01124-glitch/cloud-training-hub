## 2. ER 图描述

本系统共包含 16 张数据表，按业务模块划分为 9 个模块：

```
模块一（认证与权限）：  roles ── 1:N ── users ── 1:N ── login_logs
模块二（课程与项目）：  courses ── 1:N ── projects
模块三（小组管理）：    projects ── 1:N ── teams ── 1:N ── team_members
                        users ── 1:N ── team_members
模块四（任务看板）：    projects ── 1:N ── tasks
                        teams ── 1:N ── tasks
                        users ── 1:N ── tasks
                        bugs ── 1:1 ── tasks (related_bug_id)
模块五（日报）：        users ── 1:N ── daily_reports
                        teams ── 1:N ── daily_reports
模块六（Bug跟踪）：     users ── 1:N ── bugs (reporter_id)
                        users ── 1:N ── bugs (assignee_id)
模块七（爬虫）：        users ── 1:N ── crawler_configs ── 1:N ── crawler_data
模块八（AI记录）：      users ── 1:N ── ai_records
                        tasks ── 1:N ── ai_records
模块九（评分与统计）：  projects ── 1:N ── scores
                        teams ── 1:N ── scores
                        users ── 1:N ── scores (student_id / scored_by)
辅助模块：              users ── 1:N ── system_logs
                        users ── 1:N ── announcements
```

## 4. 关系描述

### 4.1 核心关系

- `roles` 与 `users`：一对多，一个角色下有多个用户。
- `users` 与 `login_logs`：一对多，一个用户有多条登录记录。
- `courses` 与 `projects`：一对多，一个课程下有多个实训项目。
- `projects` 与 `teams`：一对多，一个项目下有多个小组。
- `teams` 与 `team_members`：一对多，一个小组有多个成员。
- `users` 与 `team_members`：一对多，一个用户可以加入多个小组（不同项目）。
- `projects` 与 `tasks`：一对多，一个项目下有多个任务。
- `teams` 与 `tasks`：一对多，一个小组有多个任务。
- `users` 与 `tasks`（assignee）：一对多，一个用户被分配多个任务。
- `bugs` 与 `tasks`（related_bug_id）：一对一，一个 Bug 关联一个任务。
- `users` 与 `daily_reports`：一对多，一个用户有多份日报。
- `teams` 与 `daily_reports`：一对多，一个小组有多份日报。
- `crawler_configs` 与 `crawler_data`：一对多，一个配置有多条采集数据。
- `tasks` 与 `ai_records`：一对多，一个任务有多个 AI 使用记录。
- `users` 与 `ai_records`：一对多，一个用户有多条 AI 记录。
- `projects`、`teams`、`users` 与 `scores`：多对一，评分关联项目、小组和学生。

### 4.2 外键约束策略

| 约束类型 | 策略 | 说明 |
|----------|------|------|
| 外键删除 | ON DELETE CASCADE | 主表删除时级联删除子表 |
| 外键置空 | ON DELETE SET NULL | 主表删除时将外键置为 NULL |
| 联合唯一 | UNIQUE KEY | team_members(team_id,user_id)、daily_reports(user_id,report_date) |
| 数据唯一 | UNIQUE KEY | crawler_data(data_hash) 基于 SHA256 去重 |
