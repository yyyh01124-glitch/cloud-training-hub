## 2. 项目背景

### 2.1 背景分析

当前高校云计算实训课程普遍面临以下问题：

- **管理分散**：课程安排、项目任务、小组协作分散在多个平台（QQ群、飞书、Excel），信息割裂严重。
- **过程难追踪**：教师无法实时掌握每个学生的实训进度，日报提交靠人工催收，缺乏统一入口。
- **Bug与任务脱节**：学生发现问题后通过聊天工具反馈，缺乏规范的 Bug 提交流程和状态跟踪。
- **AI使用无法量化**：学生在开发过程中大量使用 AI 工具辅助编码，但实训评价体系未考虑这一因素。
- **考核数据缺失**：实训结束时缺乏可量化的过程数据支撑评分，导致评分偏主观。

### 2.2 建设目标

云智实训管理平台旨在打造一个覆盖**课程管理、项目实训、团队协作、任务跟踪、过程记录、AI辅助开发记录**的一站式实训管理平台，实现以下目标：

1. 提供统一的实训课程与项目管理入口。
2. 支持学生自选/创建小组，组内分工明确。
3. 通过任务看板实现实训任务的可视化管理。
4. 支持每日日报提交与教师点评。
5. 提供 Bug 跟踪功能，规范问题反馈与修复流程。
6. 集成 Python 爬虫数据采集，辅助实训数据积累。
7. 记录学生使用 AI 工具辅助开发的全过程。
8. 通过数据仪表盘提供多维度统计视图。

## 4. 功能模块概述

### 4.1 模块一：用户认证与权限管理（auth）

用户注册、登录、登出、个人信息编辑、密码修改、登录失败锁定。管理员可管理所有用户账号状态与角色分配。

### 4.2 模块二：课程与项目管理（projects）

教师创建课程（如"云计算应用开发实训"），在课程下创建实训项目。项目包含技术栈、评分规则、部署地址、Git仓库地址等字段。项目状态包括：未开始、进行中、已完成、已归档。

### 4.3 模块三：小组管理（teams）

学生可以从项目下创建小组或加入已有小组。组内角色包括：项目经理、后端开发、前端开发、数据库与爬虫、部署运维。每个学生在一个项目中只能加入一个小组。

### 4.4 模块四：任务看板（tasks）

Kanban 风格的任务管理面板。任务状态流转：待开始 -> 进行中 -> 待测试 -> 已完成 / 已延期 -> 已关闭。支持按项目、小组、负责人筛选。每个任务可关联 Bug。

### 4.5 模块五：每日实训日报（reports）

学生每日提交实训日报，内容涵盖：完成内容、遇到问题、AI工具使用、代码提交记录、明日计划、自评分数。教师可查看全班日报并进行点评，标记优秀日报。每人每天只能提交一份日报。

### 4.6 模块六：Bug 与问题管理（bugs）

全流程 Bug 跟踪系统。状态流转：新建 -> 已确认 -> 修复中 -> 已修复 -> 已关闭。支持按严重程度（致命/严重/一般/轻微/建议）分类。每个 Bug 可指定负责人和关联任务。

### 4.7 模块七：Python 爬虫数据采集（crawler）

可配置的爬虫采集任务管理。支持配置数据源 URL、关键词、定时表达式。采集结果去重存储，便于实训数据分析。

### 4.8 模块八：AI 辅助开发记录（ai_records）

学生记录使用 AI 工具辅助开发的过程。包含：工具名称、使用场景、提示词、AI 输出摘要、是否直接采用、修改说明、效果评价。场景分类覆盖数据库设计、Flask 路由、报错解释、样式优化、测试用例、Dockerfile 等。

### 4.9 模块九：数据统计与仪表盘（dashboard）

教师仪表盘展示：项目/小组数量、任务完成率、Bug 分布、日报提交统计。学生仪表盘展示：个人任务进度、日报提交次数、Bug 提交数、AI 使用统计。图表使用 ECharts 渲染。

## 5. 非功能性需求

| 需求类别 | 要求 |
|----------|------|
| 响应时间 | 页面加载 < 2秒，API 响应 < 500ms |
| 并发支持 | 支持 100 人同时在线 |
| 安全性 | 密码 bcrypt 加密、CSRF 保护、登录失败锁定（5次） |
| 兼容性 | 支持 Chrome/Firefox/Edge 最新版本 |
| 可维护性 | 使用 Flask Blueprint 模块化组织 |
| 部署方式 | Docker Compose 一键部署 |
