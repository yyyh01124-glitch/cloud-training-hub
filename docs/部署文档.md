## 3. 部署步骤

### 3.1 环境准备

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | bash -s docker
sudo usermod -aG docker $USER
# 重新登录使组生效

# 2. 安装 Docker Compose
sudo apt-get update
sudo apt-get install -y docker-compose-plugin

# 3. 验证安装
docker --version
docker compose version
```

### 3.2 获取代码

```bash
# 克隆项目
git clone https://github.com/yyyh01124-glitch/cloud-training-hub.git
cd cloud-training-hub

# 切换到稳定分支
git checkout main
```

### 3.3 配置环境变量

检查 `docker-compose.yml` 中的环境变量配置：

```yaml
environment:
  - FLASK_CONFIG=production
  - DATABASE_URL=mysql+pymysql://root:cth2024root@db:3306/cloud_training_hub
  - SECRET_KEY=cloud-hub-prod-secret-2026
```

生产环境建议修改 `SECRET_KEY` 为随机字符串。

### 3.4 启动服务

```bash
# 构建并启动所有服务
docker compose up -d --build

# 查看服务状态
docker compose ps
```

预期输出四个容器均显示 `Up` 状态：

```
NAME              SERVICE   STATUS   PORTS
cloudhub-web      web       Up       5000/tcp
cloudhub-db       db        Up       0.0.0.0:3307->3306/tcp
cloudhub-redis    redis     Up       0.0.0.0:6379->6379/tcp
cloudhub-nginx    nginx     Up       0.0.0.0:80->80/tcp
```

### 3.5 数据库初始化

`schema.sql` 已自动挂载到 MySQL 容器的 `/docker-entrypoint-initdb.d/` 目录，首次启动时自动执行建表脚本。

如需手动初始化：

```bash
# 进入数据库容器
docker exec -it cloudhub-db mysql -uroot -pcth2024root

# 执行建表脚本
mysql> source /docker-entrypoint-initdb.d/01-schema.sql;

# 验证表创建
mysql> USE cloud_training_hub;
mysql> SHOW TABLES;
```

## 4. Nginx 配置说明

```nginx
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    upstream flask_app {
        server web:5000;          # Docker 内部服务发现
    }

    server {
        listen 80;
        server_name _;

        client_max_body_size 16M;  # 文件上传大小限制

        # 静态文件直接由 Nginx 处理
        location /static/ {
            alias /app/static/;
            expires 7d;
            add_header Cache-Control "public, immutable";
        }

        # 上传文件访问
        location /uploads/ {
            alias /app/uploads/;
        }

        # 其他请求反向代理到 Flask
        location / {
            proxy_pass http://flask_app;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

## 5. 阿里云 ECS 配置

### 5.1 安全组规则

| 方向 | 协议 | 端口 | 授权对象 | 说明 |
|------|------|------|----------|------|
| 入方向 | TCP | 22 | 0.0.0.0/0 | SSH |
| 入方向 | TCP | 80 | 0.0.0.0/0 | HTTP |
| 入方向 | TCP | 443 | 0.0.0.0/0 | HTTPS（可选） |
| 入方向 | TCP | 3307 | 指定IP | 数据库远程访问（可选） |

### 5.2 系统优化

```bash
# 设置时区
sudo timedatectl set-timezone Asia/Shanghai

# 优化系统参数
cat >> /etc/sysctl.conf << 'EOF'
net.core.somaxconn = 65535
vm.swappiness = 10
EOF
sudo sysctl -p

# 配置防火墙（如果启用）
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw --force enable
```

### 5.3 Docker 资源限制

建议在 `docker-compose.yml` 中添加资源限制：

```yaml
services:
  web:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G

  db:
    deploy:
      resources:
        limits:
          memory: 2G
```

## 6. 常用命令

### 6.1 服务管理

```bash
# 启动所有服务
docker compose up -d

# 停止所有服务
docker compose down

# 重启所有服务
docker compose restart

# 重启单个服务
docker compose restart web

# 查看运行状态
docker compose ps

# 查看实时日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f web
```

### 6.2 数据库操作

```bash
# 进入 MySQL 容器
docker exec -it cloudhub-db mysql -uroot -pcth2024root cloud_training_hub

# 导出数据库备份
docker exec cloudhub-db mysqldump -uroot -pcth2024root cloud_training_hub > backup_$(date +%Y%m%d).sql

# 导入数据库备份
cat backup.sql | docker exec -i cloudhub-db mysql -uroot -pcth2024root cloud_training_hub
```

### 6.3 日志查看

```bash
# Flask 应用日志
docker logs cloudhub-web
docker logs -f --tail 100 cloudhub-web

# 访问日志（位于容器内部）
docker exec cloudhub-web tail -f logs/access.log

# 错误日志
docker exec cloudhub-web tail -f logs/error.log
```

### 6.4 备份与恢复

```bash
# 完整备份脚本
#!/bin/bash
BACKUP_DIR="/backup/cloudhub"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 备份数据库
docker exec cloudhub-db mysqldump -uroot -pcth2024root \
  --all-databases --single-transaction --routines --triggers \
  > $BACKUP_DIR/db_$DATE.sql

# 备份上传文件
docker cp cloudhub-web:/app/uploads $BACKUP_DIR/uploads_$DATE

# 压缩备份
tar -czf $BACKUP_DIR/cloudhub_backup_$DATE.tar.gz \
  -C $BACKUP_DIR db_$DATE.sql \
  -C $BACKUP_DIR uploads_$DATE

# 保留最近30天备份，删除旧备份
find $BACKUP_DIR -name "cloudhub_backup_*.tar.gz" -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR/cloudhub_backup_$DATE.tar.gz"
```

### 6.5 更新部署

## 7. 故障排查

### 7.1 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 容器无法启动 | 端口冲突 | `docker compose down` 后检查端口占用 |
| 数据库连接失败 | 密码不匹配 | 检查 docker-compose.yml 和 schema.sql 中的密码 |
| 502 Bad Gateway | Gunicorn 未启动 | `docker logs cloudhub-web` 查看错误 |
| 静态资源404 | Nginx 路径配置错误 | 检查 nginx.conf 中的 alias 路径 |
| 文件上传失败 | 权限不足 | `docker exec cloudhub-web chmod 755 /app/uploads` |

### 7.2 健康检查

```bash
# 检查所有容器健康状态
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# 检查 Flask 应用健康
curl -I http://localhost

# 检查数据库连接
docker exec cloudhub-web python -c "
from app import create_app
from app.extensions import db
app = create_app()
with app.app_context():
    db.session.execute('SELECT 1')
    print('Database connection OK')
"
```
