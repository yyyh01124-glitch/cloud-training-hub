## 2. 项目概述

### 2.1 项目背景

云智实训管理平台是面向高校云计算实训课程的全流程管理平台。在实训教学过程中，教师需要管理多个课程和实训项目，学生需要以小组协作方式完成项目开发任务。传统的实训管理方式依赖 Excel、QQ 群、飞书等多工具组合，存在信息分散、过程难追踪、考核数据缺失等问题。

本项目的目标是打造一个**覆盖实训全流程**的管理平台，包含课程项目发布、小组协作、任务看板、日报提交、Bug跟踪、AI辅助开发记录等功能，实现实训过程的数字化管理。

### 2.2 项目成果

经过约 4 个月的开发，项目已成功完成第一版交付：

- **代码规模**：后端 Python 代码约 3000 行，前端模板 40+ 页面
- **数据库**：16 张 MySQL 表，覆盖 9 个业务模块
- **部署方式**：Docker Compose 一键部署
- **上线地址**：http://8.146.230.215（阿里云 ECS）
- **代码仓库**：https://github.com/yyyh01124-glitch/cloud-training-hub

## 4. 技术亮点

### 4.1 模块化蓝图架构

采用 Flask Blueprint 将系统拆分为 10 个独立蓝图，每个蓝图对应一个业务模块。模块间通过 models.py 中的外键关系解耦，路由通过 url_prefix 隔离。这种架构使代码组织清晰、易于扩展和维护。

### 4.2 RBAC 权限控制

基于装饰器的权限控制方案，只需在路由函数上添加 `@role_required('admin', 'teacher')` 即可实现精确到角色的访问控制。配合 Flask-Login 的 Session 管理，实现全站统一的认证授权体系。

### 4.3 状态机设计

任务和 Bug 模块各自实现了完整的有限状态机。状态转换规则在代码层严格校验，非法转换会被拒绝。例如 Bug 不能从 "新建" 直接跳到 "已修复"，必须经过 "已确认" -> "修复中" 的完整流转路径。

### 4.4 爬虫数据去重

爬虫模块使用 SHA256 哈希对采集数据进行唯一性校验，`data_hash` 字段设为 UNIQUE KEY，确保同一条数据不会被重复入库。

### 4.5 Docker 容器化

通过 Docker Compose 编排 Web、数据库、缓存、反向代理四个服务。数据库初始化脚本自动挂载到初始化目录，首次启动自动建表。生产环境使用 Gunicorn 多 workers 运行，Nginx 做反向代理和静态资源处理。

## 5. 经验教训

### 5.1 做得好的方面

1. **模块化设计充分**：前期投入时间设计蓝图结构和数据库关系，后期开发非常顺畅，模块之间几乎没有耦合问题。
2. **AI 辅助开发提效明显**：使用 AI 工具辅助生成路由代码、SQL 查询、测试用例等，整体开发效率提升了约 40%。
3. **Docker 部署一次成功**：由于前期充分测试了 Dockerfile 和 docker-compose.yml，上线部署过程非常顺利，未出现环境兼容性问题。
4. **状态机设计严谨**：Bug 状态转换的严格校验在测试阶段发现了多处逻辑问题，有效避免了数据不一致。

### 5.2 可以改进的方面

1. **前端交互体验不足**：任务看板缺少拖拽功能，用户需要点击下拉菜单选择状态，操作不够直观。计划在 V1.1 中加入 SortableJS 拖拽。
2. **测试自动化不足**：当前以手动测试为主，缺少自动化测试脚本。后续需要补充 pytest 单元测试和集成测试。
3. **错误处理不够友好**：部分异常场景（如数据库连接超时）的用户提示不够友好，需要优化全局异常处理。
4. **缺少消息通知**：任务分配、Bug 分配等操作后，被分配人没有收到通知，需要手动刷新页面查看。
5. **文档编写滞后**：部分接口文档和部署文档在开发完成后才补充，建议后续采用文档即代码的方式同步更新。
